# 设置导航顺序设计

## 目标

面向已有使用习惯的老用户，将设置侧栏按日常使用频率和功能重要度重新排序，使最常调整的角色、AI、窗口、行为与显示入口优先出现在首屏。

## 约束

- 所有用户使用同一套固定顺序，不根据个人点击记录动态变化。
- 只调整现有导航项目顺序，不新增分组标题、分隔线或其他导航样式。
- 不修改导航键、页面构建逻辑、图标、颜色、文案和页面交互。
- 默认入口仍为“角色列表”。
- “关于”继续固定在侧栏底部，不进入可滚动导航区域。

## 最终顺序

1. 角色列表
2. LLM 配置
3. 悬浮窗设置
4. 角色行为
5. 显示设置
6. TTS 配置
7. 语音识别
8. 闹钟 / 番茄钟
9. 好感度 / 记忆
10. 聊天记录
11. 屏幕感知
12. POV 设置
13. 记忆相册
14. 数据统计
15. 聊天接入
16. 工具与电脑控制
17. 数据管理
18. 关系教程

“关于”保持独立并固定在底部。

## 实现范围

修改 `settings_window/settings_window.py` 中 `_build_sidebar` 创建并加入导航按钮的顺序。按钮仍写入原有的 `self._nav_buttons` 键，并继续连接 `_on_nav_selected`，因此懒加载页面、选中状态、滚动定位和指示器逻辑均保持不变。

不引入导航配置表、点击统计或动态排序逻辑。当前需求只有一个固定顺序，直接沿用现有按钮构建模式更清晰，变更范围也最小。

## 验证标准

- 可滚动导航区按最终顺序显示全部 18 个入口。
- 打开设置窗口时仍默认选中“角色列表”。
- 每个入口仍能切换到对应页面。
- 导航图标、颜色和文案与调整前一致。
- “关于”仍位于侧栏底部且不随可滚动区域移动。
- Python 语法检查通过。

## 非目标

- 不记录或分析用户点击频率。
- 不支持用户自定义排序。
- 不调整页面内部控件顺序。
- 不重构设置窗口或导航组件。
